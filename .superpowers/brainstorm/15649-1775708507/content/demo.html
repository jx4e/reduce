Binary Search Trees
Data Structures — Week 4
4 sections · ~12 min read
1
What is a BST?
▼

A binary search tree is a node-based data structure where each node has at most two children. For any node, all values in the left subtree are smaller, and all values in the right subtree are larger.

BST property: left.val < node.val < right.val
2
Search & Insertion
▶

Search starts at the root and compares the target value: go left if smaller, right if larger. Insertion follows the same path and adds the new node at the correct leaf position. Both operations run in O(h) time where h is the tree height.

3
Time Complexity
▶

Average case: O(log n) for search, insert, delete. Worst case (degenerate tree): O(n). Balanced BSTs like AVL trees and Red-Black trees guarantee O(log n) worst case.

4
In-order Traversal
▶

In-order traversal (left → root → right) of a BST always produces values in sorted ascending order. This property makes BSTs useful for implementing sorted sets and maps.
